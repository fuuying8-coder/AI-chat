# 语音输入子模块 - 笔记

基于 **Web Speech API** 实现麦克风录制控制、语音转写、状态提示，用于优化语音对话体验。

---

## 1. 代码结构

```
src/
├── voice/
│   ├── index.js           # 模块入口，导出 useVoiceInput、VoiceStatus
│   └── useVoiceInput.js   # 语音 composable：识别实例、状态、起停与转写
└── components/
    └── ChatInput.vue      # 使用 useVoiceInput，麦克风按钮、状态区、转写填入输入框
```

- **`useVoiceInput.js`**：纯逻辑 composable，不依赖 UI，可在任意输入场景复用。
- **`ChatInput.vue`**：集成语音按钮、状态提示、录音中 placeholder、错误提示；在 `processing` 时把转写文本追加到输入框并 `reset()`。

### 1.2 `src/voice/useVoiceInput.js`

```js
/**
 * 语音输入 composable：基于 Web Speech API
 * 功能：麦克风录制控制、语音转写、状态提示（idle/listening/processing/error）
 */

import { ref, computed, onUnmounted } from 'vue'

const SpeechRecognition =
  typeof window !== 'undefined' &&
  (window.SpeechRecognition || window.webkitSpeechRecognition)

export const VoiceStatus = {
  IDLE: 'idle',
  LISTENING: 'listening',
  PROCESSING: 'processing',
  ERROR: 'error',
}

const STATUS_LABELS = {
  [VoiceStatus.IDLE]: '',
  [VoiceStatus.LISTENING]: '正在聆听...',
  [VoiceStatus.PROCESSING]: '识别中...',
  [VoiceStatus.ERROR]: '识别失败',
}

const ERROR_MESSAGES = {
  'not-allowed': '请允许麦克风权限',
  'no-speech': '未检测到语音，请重试',
  'audio-capture': '无法访问麦克风',
  'network': '网络错误，请检查连接',
  'aborted': '已取消',
  'language-not-supported': '当前浏览器不支持该语言',
  default: '语音识别失败，请重试',
}

/**
 * @param {Object} options
 * @param {string} [options.lang='zh-CN'] 识别语言
 * @param {boolean} [options.continuous=true] 是否连续识别
 * @param {boolean} [options.interimResults=true] 是否返回临时结果
 * @param {number} [options.maxAlternatives=1] 最大备选结果数
 */
export function useVoiceInput(options = {}) {
  const {
    lang = 'zh-CN',
    continuous = true,
    interimResults = true,
    maxAlternatives = 1,
  } = options

  const status = ref(VoiceStatus.IDLE)
  const transcript = ref('')
  const interimTranscript = ref('')
  const errorMessage = ref('')

  const isSupported = !!SpeechRecognition
  const isRecording = computed(() => status.value === VoiceStatus.LISTENING)
  const statusLabel = computed(() => STATUS_LABELS[status.value] || '')
  const hasError = computed(() => status.value === VoiceStatus.ERROR)

  let recognition = null

  function getRecognition() {
    if (!SpeechRecognition) return null
    if (!recognition) {
      recognition = new SpeechRecognition()
      recognition.continuous = continuous
      recognition.interimResults = interimResults
      recognition.maxAlternatives = maxAlternatives
      recognition.lang = lang

      recognition.onstart = () => {
        status.value = VoiceStatus.LISTENING
        transcript.value = ''
        interimTranscript.value = ''
        errorMessage.value = ''
      }

      recognition.onresult = (event) => {
        let finalText = ''
        let interimText = ''
        for (let i = event.resultIndex; i < event.results.length; i++) {
          const result = event.results[i]
          const text = result[0].transcript
          if (result.isFinal) {
            finalText += text
          } else {
            interimText += text
          }
        }
        if (finalText) transcript.value += finalText
        interimTranscript.value = interimText
      }

      recognition.onend = () => {
        if (status.value === VoiceStatus.LISTENING) {
          status.value = VoiceStatus.PROCESSING
        }
      }

      recognition.onerror = (event) => {
        status.value = VoiceStatus.ERROR
        errorMessage.value = ERROR_MESSAGES[event.error] || ERROR_MESSAGES.default
        if (event.error === 'aborted') {
          status.value = VoiceStatus.IDLE
        }
      }
    }
    return recognition
  }

  function start() {
    if (!isSupported) {
      status.value = VoiceStatus.ERROR
      errorMessage.value = '当前浏览器不支持语音识别，请使用 Chrome 或 Edge'
      return
    }
    const rec = getRecognition()
    if (!rec) return
    try {
      rec.start()
    } catch (e) {
      status.value = VoiceStatus.ERROR
      errorMessage.value = e.message || ERROR_MESSAGES.default
    }
  }

  function stop() {
    if (!recognition || status.value !== VoiceStatus.LISTENING) return
    try {
      recognition.stop()
    } catch (e) {
      // 忽略已停止的情况
    }
    status.value = VoiceStatus.PROCESSING
  }

  function toggle() {
    if (isRecording.value) {
      stop()
    } else {
      start()
    }
  }

  function reset() {
    status.value = VoiceStatus.IDLE
    transcript.value = ''
    interimTranscript.value = ''
    errorMessage.value = ''
  }

  /** 获取最终文本（含临时结果），用于填入输入框 */
  function getFullTranscript() {
    const t = transcript.value + interimTranscript.value
    return t.trim()
  }

  onUnmounted(() => {
    if (recognition && status.value === VoiceStatus.LISTENING) {
      try {
        recognition.abort()
      } catch (e) {}
    }
  })

  return {
    isSupported,
    status,
    transcript,
    interimTranscript,
    errorMessage,
    isRecording,
    statusLabel,
    hasError,
    start,
    stop,
    toggle,
    reset,
    getFullTranscript,
  }
}
```

## 2. Web Speech API 使用要点

- **兼容性**：`window.SpeechRecognition || window.webkitSpeechRecognition`，Chrome / Edge 支持较好，部分浏览器不支持。
- **能力**：浏览器端语音识别（通常走云端），无需自建 ASR；需麦克风权限与网络。
- **不涉及**：不直接使用 `getUserMedia` / `MediaRecorder` 做“录音文件”，只做实时语音转文字。

## 3. 状态机与状态提示

| 状态 | 常量 | 说明 | 界面提示 |
|------|------|------|----------|
| 空闲 | `VoiceStatus.IDLE` | 未录音 | 无 |
| 聆听 | `VoiceStatus.LISTENING` | 正在录音与识别 | 「正在聆听...」 |
| 处理中 | `VoiceStatus.PROCESSING` | 本次识别结束，可取最终文本 | 「识别中...」 |
| 错误 | `VoiceStatus.ERROR` | 权限/网络/无语音等 | 见下方错误文案 |

错误码与文案（`ERROR_MESSAGES`）：

- `not-allowed` → 请允许麦克风权限
- `no-speech` → 未检测到语音，请重试
- `audio-capture` → 无法访问麦克风
- `network` → 网络错误，请检查连接
- `aborted` → 已取消（并回到 IDLE）
- `language-not-supported` → 当前浏览器不支持该语言
- 默认 → 语音识别失败，请重试

## 4. useVoiceInput API

### 4.1 选项

```js
useVoiceInput({
  lang: 'zh-CN',        // 识别语言
  continuous: true,     // 连续识别（多段话不自动停）
  interimResults: true, // 返回临时结果（边说边出字）
  maxAlternatives: 1,   // 备选结果数量
})
```

### 4.2 返回值

| 属性/方法 | 类型 | 说明 |
|-----------|------|------|
| `isSupported` | `boolean` | 当前环境是否支持语音识别 |
| `status` | `Ref<string>` | 当前状态（idle/listening/processing/error） |
| `transcript` | `Ref<string>` | 已确认的转写文本 |
| `interimTranscript` | `Ref<string>` | 当前临时结果 |
| `errorMessage` | `Ref<string>` | 错误提示文案 |
| `isRecording` | `ComputedRef<boolean>` | 是否正在录音 |
| `statusLabel` | `ComputedRef<string>` | 当前状态对应中文提示 |
| `hasError` | `ComputedRef<boolean>` | 是否处于错误状态 |
| `start()` | `function` | 开始录音/识别 |
| `stop()` | `function` | 停止录音，进入 processing |
| `toggle()` | `function` | 录音 ⇄ 停止 切换 |
| `reset()` | `function` | 清空状态与文本，回到 idle |
| `getFullTranscript()` | `function` | 获取「最终 + 临时」拼接后的全文，用于填入输入框 |

## 5. ChatInput 中的交互与优化

- **麦克风按钮**  
  - 不支持时：禁用 + tooltip「当前浏览器不支持语音输入」。  
  - 支持时：tooltip 为「语音输入」/「点击停止录音」；录音中 `is-recording` 红色脉冲样式。  
  - 生成中（`loading`）：禁用语音按钮，避免与回复冲突。

- **状态区**  
  - `voice.statusLabel` 或 `voice.errorMessage` 存在时显示一行提示；错误用 `.voice-error` 红色，正常用 `.voice-hint` 蓝色。

- **转写填入输入框**  
  - 使用 `watch(voice.status)`，当变为 `processing` 时：  
    - 取 `voice.getFullTranscript()`，非空则追加到 `inputValue`（已有内容则先换行再追加）。  
    - 调用 `voice.reset()` 回到 idle，便于下次录音。

- **卸载**  
  - composable 内 `onUnmounted` 时若正在聆听会 `recognition.abort()`，避免泄漏与后台继续录音。

### 5.1 ChatInput 语音相关代码

**Script：引入与状态监听、点击处理**

```js
import { useVoiceInput } from '@/voice/useVoiceInput'

const inputValue = ref('')

// 语音输入
const voice = useVoiceInput({ lang: 'zh-CN', continuous: true, interimResults: true })
watch(voice.status, (status) => {
  if (status === 'processing') {
    const text = voice.getFullTranscript()
    if (text) {
      inputValue.value = (inputValue.value ? inputValue.value + '\n' : '') + text
    }
    voice.reset()
  }
})

// 语音输入：生成中禁用；点击切换录音
const handleVoiceToggle = () => {
  if (props.loading) return
  if (!voice.isSupported) return
  voice.toggle()
}
```

## 6. 流程简述

1. 用户点击麦克风 → `handleVoiceToggle()` → `voice.toggle()`。
2. 首次 start → `getRecognition()` 创建 `SpeechRecognition`，配置 `lang/continuous/interimResults`，绑定 `onstart/onresult/onend/onerror`。
3. `onstart` → status = listening，清空 transcript/interim/error。
4. `onresult` → 按 `isFinal` 累加最终/临时文本到 `transcript` / `interimTranscript`。
5. 用户点击停止或识别结束 → `onend` → status = processing（若当前为 listening）。
6. ChatInput 的 watch 发现 processing → 取 `getFullTranscript()` 追加到输入框 → `voice.reset()`。
7. 出错时 `onerror` → status = error，设置 `errorMessage`；若为 `aborted` 则回到 idle。

## 7. 可扩展点

- **多语言**：通过 `useVoiceInput({ lang: 'en-US' })` 等切换。
- **仅最终结果**：`interimResults: false`，不显示“边说边出字”。
- **在其它组件复用**：直接 `import { useVoiceInput } from '@/voice'`，按需绑定到输入框或自定义 UI。
- **与后端 ASR 并存**：当前为纯前端 Web Speech API；若需自建 ASR，可在此 composable 外再包一层“识别源”抽象（浏览器 vs 后端），内部仍复用同一套状态与 UI 逻辑。
